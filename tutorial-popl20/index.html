Tutorial on Proving Semantic Type Soundness in Iris

20 January 2020, at POPL, New Orleans, USA
Coq Formalization of Iris Tutorial repository Tutorial info on POPL'20 website

In this tutorial we demonstrate how the Iris framework for concurrent separation logic can be used to prove type soundness of feature-rich, realistic languages, using the "semantic approach'' to type soundness. Contrary to the traditional "syntactic approach" of progress and preservation, the semantic approach has several advantages in terms of reasoning about data abstraction and unsafe features. We show that the semantic approach leads to clear and concise proofs that can be effectively mechanized using the implementation of Iris in the Coq proof assistant.

Tutorial Material

The tutorial material can be found in the Tutorial repository at https://gitlab.mpi-sws.org/iris/tutorial-popl20. Below is an overview of the files in that repository with links to coqdoc generated html documentation of these files.

  • base.v: Some basic definitions and notations.
  • types.v: Syntactic types.
  • typed.v: Syntactic typing rules.
  • sem_types.v: Semantic types.
  • sem_type_formers.v: Type formers, e.g., product types, sum types, function types, etc., for semantic types.
  • sem_operators.v: Boilerplate to support typing operators, e.g., +, -, *, etc.
  • sem_typed.v: The semantic typing relation for typing contexts and (based on that) open programs.
  • safety.v: The adequacy theorem for our semantic types, i.e., any semantically typed program is safe.
  • compatibility.v: Compatibility lemmas.
  • interp.v: Interpretation of syntactic types using semantic type formers.
  • fundamental.v: The fundamental theorem of logical relations.
  • unsafe.v: Examples of programs that are not syntactically well-typed but are semantically typed. (Includes solution to exercises)
  • parametricity.v: Examples of using our logical relations to prove (unary) parametricity theorems. (Includes solution to exercises)
